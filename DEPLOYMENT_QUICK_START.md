# Quick Deployment Guide

## üöÄ Django Deployment on Render

### 1. Quick Steps
1. **Sign up at render.com** and connect your GitHub repository
2. **Create Web Service**: Choose your repo, select `Python` runtime
3. **Configure Build**: 
   - Build Command: `pip install --upgrade pip && pip install -r requirements.txt && python manage.py collectstatic --noinput`
   - Start Command: `bash start.sh`
4. **Create PostgreSQL Database**: Name it `wolvcapital-db`
5. **Set Environment Variables** (see detailed list below)
6. **Deploy and test**: Check `your-app.onrender.com/healthz/`

### 2. Essential Environment Variables for Render
```bash
SECRET_KEY=<click-generate-button>
DEBUG=0
DJANGO_SETTINGS_MODULE=wolvcapital.settings
ALLOWED_HOSTS=api.wolvcapital.com,your-app-name.onrender.com
CORS_ALLOWED_ORIGINS=https://wolvcapital.com,https://www.wolvcapital.com,https://api.wolvcapital.com
CSRF_TRUSTED_ORIGINS=https://wolvcapital.com,https://www.wolvcapital.com,https://api.wolvcapital.com
WEB_CONCURRENCY=2
```

## üåê Custom Domain Setup

### 3. Domain Architecture
- `wolvcapital.com` ‚Üí Vercel (Next.js Frontend) 
- `www.wolvcapital.com` ‚Üí Vercel (redirects to apex)
- `api.wolvcapital.com` ‚Üí Render (Django Backend)

### 4. DNS Records to Add
At your domain registrar (GoDaddy, Namecheap, etc.):

| Type | Name | Value | Purpose |
|------|------|-------|---------|
| A | @ | [Vercel IP from dashboard] | Frontend apex |
| CNAME | www | [Vercel CNAME from dashboard] | Frontend www |
| CNAME | api | your-app-name.onrender.com | Backend API |

### 5. Get DNS Values
- **Vercel**: Dashboard ‚Üí Your Project ‚Üí Settings ‚Üí Domains ‚Üí Add domain
- **Render**: Dashboard ‚Üí Your Service ‚Üí Settings ‚Üí Custom Domains ‚Üí Add domain

## üÜò Quick Troubleshooting

### CORS Errors
- Verify CORS_ALLOWED_ORIGINS includes all three domains
- No trailing slashes in environment variables
- Redeploy backend after CORS changes

### DNS Not Working
- Wait longer (up to 48 hours max)
- Check with https://dnschecker.org
- Clear browser DNS cache

### API Calls Wrong Domain
- Check NEXT_PUBLIC_API_URL in Vercel
- Redeploy frontend after env changes
- Clear browser cache (Ctrl+Shift+R)
